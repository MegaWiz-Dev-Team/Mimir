### 3. Multi-Insurer Architecture (5 min)

**Design Decision: Single Tenant + insurer_id Field**

```
asgard_insurance tenant:
├── Prudential (insurer_001)
│   ├── 960 chunks
│   ├── language: en
│   └── products: Health, Life, Savings, Investment
├── AXA (insurer_002)
│   ├── 550 chunks
│   ├── language: bi (bilingual)
│   └── products: Auto, Home, Travel, Medical
└── Thai Health (insurer_003)
    ├── 340+ chunks
    ├── language: th
    └── products: Health, Accident, Critical Illness
```

**Why Single Tenant?**
- ✅ Cross-insurer comparison (side-by-side search)
- ✅ Simpler Mimir configuration
- ✅ Unified compliance auditing
- ✅ Easy to add new insurers (just new insurer_id)

**Deduplication Strategy:**
- Detect duplicate products across insurers (same coverage, different names)
- Flag with confidence score (0.8-1.0)
- Manual review if confidence 0.7-0.8
- Impact: ~3-5% of chunks will be marked duplicates

### 4. New Features (10 min)

#### Feature 1: File Upload + OCR

**Supported Formats:**
- 📄 PDF (via pypdf)
- 📝 DOCX (via python-docx)
- 📋 TXT (plain text)
- 🖼️ Images (JPG, PNG via Syn OCR + pytesseract fallback)

**Implementation:**
```python
# Phase 1 will support:
python main.py --phase 1 --upload-dir data/uploads/

# Processes all files, extracts text/OCR, creates chunks
# Output: phase1_chunks.jsonl with "source_type": "upload"
```

**OCR Pipeline:**
1. Check if Syn service available
2. Send image to Syn (/ocr endpoint)
3. Fallback to pytesseract if Syn fails
4. Confidence threshold: 80%

#### Feature 2: Thai Language Support

**Thai Preprocessing:**
```
Thai text → pythainlp tokenization → Thai NER → Entities
```

**Thai NER Entities:**
- Products (ประกันภัย)
- Coverage types (ความคุ้มครอง)
- Medical conditions (โรค)
- Exclusions (ข้อยกเว้น)

**Language Tagging:**
- All chunks tagged with "language": "en" | "th" | "bi"
- Phase 3 applies language-specific NER
- Phase 5 evaluates Hit Rate separately by language

**Test Queries (Thai):**
```
"ความคุ้มครองโรคร้ายแรง" (Critical illness coverage)
"บัตรประกันสุขภาพ" (Health card)
"ค่าใช้จ่ายเบื้องต้น" (Deductible)
```

#### Feature 3: Compliance Checks (Phase 6 NEW)

**Regulatory Rules (Thai OIC):**
- Coverage limits: Min/max per product type
- Exclusion consistency: No conflicting exclusions
- Transparency requirements: All exclusions listed

**Implementation:**
