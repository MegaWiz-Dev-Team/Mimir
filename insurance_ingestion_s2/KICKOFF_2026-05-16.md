### 2. Architecture & Design System (5 min)

**Core Patterns (core.py):**
- `PipelineConfig`: 16 immutable settings (Mimir, Neo4j, Qdrant, Heimdall)
- `PipelineLogger`: Phase-aware output (✅ success, ⚠️ warning, ❌ error)
- `Chunk` + `Entity`: Strongly-typed data classes with metadata
- Domain-specific exceptions: IngestionError, EmbeddingError, ValidationError

**Five Phases (TDD-First):**
```
Phase 1 (Data Eng):   Extract products from URLs → 960 chunks
Phase 2 (Backend):    Normalize schema, abstract PII → 21 metadata keys
Phase 3 (Backend):    Extract entities, build knowledge graph → 500+ entities
Phase 4 (Backend+DevOps): Ingest to Mimir, embed, index Qdrant
Phase 5 (QA):         Validate search quality → Hit Rate@3 ≥75%
```

**Why This Design?**
- TDD forces test coverage (20+ tests ready)
- Design system prevents inconsistency (all phases use same logger, config, data classes)
- Decision gates (May 22, May 27) align with fallback strategy
