# 📖 คำอธิบายประกอบ: Business Requirement Document (BRD)
## โปรเจกต์ Project-Mimir (Ragnarok Online: AI-Native Evolution)

> [!NOTE]
> **Project-Mimir (มิเมียร์):** ในตำนานนอร์ส Mimir คือเทพแห่งความรู้และสติปัญญา และเป็นผู้รักษาบ่อน้ำศักดิ์สิทธิ์ (Mímisbrunnr) ซึ่งเปรียบเสมือนระบบ RAG (Oracle Bot) ที่ทำหน้าที่เป็นคลังความรู้ของเซิร์ฟเวอร์ที่เรากำลังพัฒนา

> เอกสารฉบับนี้เป็น **คำอธิบายภาษาไทย** ประกอบเอกสาร [BRD_RO-AI.md](file:///Volumes/T7%20Shield/Project-Mimir/BRD_RO-AI.md) เพื่อให้ทีมงานเข้าใจบริบทของแต่ละส่วนได้ง่ายขึ้น

## 1. บทสรุปผู้บริหาร (Executive Summary)

**ปัญหาที่โปรเจกต์นี้แก้:** เกม Ragnarok Online แบบดั้งเดิมมี NPC ที่พูดซ้ำ ๆ เหมือนกันทุกครั้ง ไม่มีชีวิตชีวา ทำให้ผู้เล่นเบื่อเร็ว นอกจากนี้ การดูแลเซิร์ฟเวอร์โดย GM ต้องใช้คนจริงตลอด 24 ชั่วโมง ซึ่งมีค่าใช้จ่ายสูง

**วิธีแก้:** ใช้ **AI (Large Language Model)** มาทำให้ NPC พูดคุยได้อัจฉริยะ มี Bot ช่วยตอบคำถาม มี GM อัตโนมัติ และมีระบบหารายได้ผ่าน AI — ทั้งหมดรันบน **Mac mini M4 Pro (64GB RAM)** ที่เรามีอยู่แล้ว ช่วยประหยัดค่า Cloud ในช่วงเริ่มต้น

## 3. User Stories — คำอธิบายแต่ละโมดูล

### 🏰 Module A: Generative NPC ("โลกที่มีชีวิต")

**แนวคิดหลัก:** NPC ไม่พูดข้อความตายตัวอีกต่อไป แต่ใช้ AI สร้างบทสนทนาใหม่ทุกครั้ง ตอบตามบุคลิกที่กำหนดไว้

**ตัวอย่างสถานการณ์:**
- ผู้เล่น Level 10 ถาม NPC Sage → NPC จะตอบบทเรียนพื้นฐาน ให้กำลังใจ
- ผู้เล่น Level 99 ถาม NPC เดิม → NPC จะตอบแบบลึกซึ้ง ท้าทายมากกว่า
- NPC ไม่ใช่แค่ "พูด" แต่สามารถ **กระทำ** ได้ เช่น Heal ผู้เล่น, Buff, หรือให้ Quest Item

**ข้อกำหนดสำคัญ:**
- ⏱ ตอบภายใน **2 วินาที** (ไม่งั้นผู้เล่นจะรู้สึกว่าเกมค้าง)
- 🔄 ถ้า AI ล่ม NPC ต้อง **สลับกลับไปใช้ Script เดิม** อัตโนมัติ (ผู้เล่นจะไม่สังเกตเห็น)

### 🔮 Module B: Oracle RAG Bot ("พี่เลี้ยงอัจฉริยะ")

**แนวคิดหลัก:** Bot ช่วยตอบคำถามเกี่ยวกับเกมที่ดึงข้อมูลจาก **ฐานข้อมูลจริงของเซิร์ฟเวอร์** ไม่ใช่ข้อมูลจาก Wiki ทั่วไป

**ทำไมถึงสำคัญ:**
- เซิร์ฟเวอร์แต่ละตัวมี Rate/Drop/Config ไม่เหมือนกัน
- ข้อมูลจากเว็บนอกอาจผิดสำหรับเซิร์ฟเวอร์นี้
- ใช้เทคโนโลยี **RAG (Retrieval-Augmented Generation)** = AI ค้น Vector DB ก่อน แล้วตอบจากข้อมูลที่หามาได้

**ตัวอย่างคำถามที่ตอบได้:**
- "การ์ดอะไรดีที่สุดสำหรับ Knight ไปตี High Orc?"
- "มอนสเตอร์ตัวไหน Drop กล่อง Old Blue Box?"
- "อยากเก็บเลเวลเร็ว ๆ ควรไปแมพไหน?"

### 🤖 Module C: AI Game Master ("GM ที่ไม่เคยหลับ")

**แนวคิดหลัก:** AI คอยดูแลเซิร์ฟเวอร์ตลอด 24/7 ทำสองอย่างหลัก:

1. **จับ Bot/Hack:** วิเคราะห์ Log แบบ Real-time หาพฤติกรรมผิดปกติ
   - เดินซ้ำรูปแบบเดิมตลอด → น่าจะ Bot
   - เคลื่อนที่เร็วผิดปกติ → น่าจะ Speed Hack

2. **สร้าง Event อัตโนมัติ:** ถ้าผู้เล่นออนไลน์น้อยลง AI จะ:
   - Spawn Boss เซอร์ไพรส์
   - เพิ่ม Drop Rate ชั่วคราว
   - ประกาศ Event กระตุ้นให้คนกลับมาเล่น

### 💰 Module D: ระบบหารายได้ด้วย AI

**3 ฟีเจอร์หารายได้:**

| ฟีเจอร์                | คำอธิบาย                             | รูปแบบรายได้             |
| -------------------- | ---------------------------------- | ---------------------- |
| **Smart Homunculus** | สัตว์เลี้ยงที่คุยกลยุทธ์กับผู้เล่นได้            | สมาชิก VIP (รายเดือน)    |
| **Fortune Teller**   | NPC ดูดวงประจำวัน ให้ Buff/Item ตามดวง | ซื้อ Token กดดวง (Gacha) |
| **AI Support**       | ระบบช่วยเหลือผู้เล่นอัตโนมัติ              | ลดค่าใช้จ่าย GM           |

## 4. โมเดลรายได้ (Revenue Model)

**3 ระดับ VIP ที่เสนอ:**

| ระดับ      | ราคา                | สิ่งที่ได้                        |
| --------- | ------------------- | ---------------------------- |
| **VIP 1** | ~100 บาท/เดือน ($3)  | ใช้ Oracle ถามข้อมูลเกมได้       |
| **VIP 2** | ~280 บาท/เดือน ($8)  | ได้ Smart Homunculus ที่คุยด้วยได้ |
| **VIP 3** | ~520 บาท/เดือน ($15) | ได้ทุกฟีเจอร์ AI                 |

> [!IMPORTANT]
> **ราคาเป็นค่าเริ่มต้น** สามารถปรับได้ตามตลาดเป้าหมาย (ไทย vs อินเตอร์)

**ระบบป้องกันเศรษฐกิจเกมพัง:**
- AI ให้ Zeny ได้ไม่เกิน **1 ล้าน/วัน** ทั้งเซิร์ฟเวอร์
- AI ให้ไอเท็มได้ไม่เกิน **5 ชิ้น/คน/วัน**
- AI ให้การ์ด MVP หรือของหายากไม่ได้เด็ดขาด
- Buff จาก Fortune Teller หมดเวลาภายใน 30 นาที

## 5. ตัวชี้วัดความสำเร็จ (KPIs)

**เป้าหมายหลักใน 6 เดือน:**

| ตัวชี้วัด                | เป้าหมาย    | ทำไม                                 |
| -------------------- | ---------- | ----------------------------------- |
| ผู้เล่นรายวัน (DAU)      | เพิ่ม +30%   | AI ทำให้เกมน่าเล่นขึ้น                    |
| อัตราคงอยู่ Day-30      | เพิ่ม +15 จุด | ผู้เล่นไม่เบื่อเร็ว เพราะ NPC พูดต่างกันทุกครั้ง |
| รายได้เฉลี่ยต่อคน (ARPU) | $5/เดือน    | ฟีเจอร์ VIP มีคุณค่าจริง                  |
| เวลาเล่นเฉลี่ย          | เพิ่ม +35%   | Oracle + NPC ทำให้อยู่ในเกมนานขึ้น        |
| ลดงาน GM             | -50%       | AI GM จัดการงาน Routine แทน          |

**ข้อตกลงระดับบริการ (SLA):**
- Chat ต้องตอบภายใน **2 วินาที** (95% ของ Request)
- ระบบต้อง Uptime **≥99.5%**
- ถ้า AI ล่ม ต้องสลับไป Static Script ภายใน **0.5 วินาที**

## 6. การประเมินความเสี่ยง (Risk Assessment)

**ความเสี่ยงที่ต้องระวังที่สุด:**

| ความเสี่ยง                       | ทำไมถึงร้ายแรง                | วิธีป้องกัน                          |
| ------------------------------ | -------------------------- | -------------------------------- |
| 🔴 AI พูดคำหยาบ/ไม่เหมาะสม         | ทำลายภาพลักษณ์ ผู้เล่นร้องเรียน    | ฟิลเตอร์ 2 ชั้น (ก่อน+หลัง AI ตอบ)     |
| 🔴 ผู้เล่นหลอก AI ให้ของ            | เศรษฐกิจเกมพัง               | ลิมิตแข็ง + Whitelist Action ต่อ NPC |
| 🟡 AI ตอบข้อมูลผิด (Hallucination) | ผู้เล่นเสียเวลาตามข้อมูลผิด       | ใช้ RAG ดึงข้อมูลจริงก่อนตอบ           |
| 🟡 AI ทำให้เกม Pay-to-Win         | ผู้เล่นฟรีไม่พอใจ หนีไปเล่นเซิร์ฟอื่น | VIP ให้แค่ความสะดวก ไม่ให้ Power     |

## 7. แผนปล่อยเวอร์ชัน (Release Phases)

```
Phase 1 (5 สัปดาห์) → เตรียม Infrastructure: Docker, DB, LLM POC
Phase 2 (7 สัปดาห์) → NPC Chat + Oracle Bot + ระบบ Safety
Phase 3 (4 สัปดาห์) → AI GM + ฟีเจอร์หารายได้
Phase 4 (4 สัปดาห์) → Load Test + Beta Test + แก้บัค
Phase 5 (4 สัปดาห์) → ย้ายขึ้น Google Cloud (Scale)
```

**Gate สำคัญ:** จะไม่ผ่าน Phase ถ้า:
- Phase 1: LLM ตอบไม่ได้ภายใน 2 วินาที → ต้องแก้ก่อน
- Phase 4: รองรับ 200 คนพร้อมกันไม่ได้ → ต้อง Optimize ก่อน
