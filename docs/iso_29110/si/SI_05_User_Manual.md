# SI-05: User Manual (คู่มือการใช้งาน)
**Project Name:** Project Mimir
**Sprint:** 7 (Final - Documenting Complete Flow)

## 1. System Overview (ภาพรวมระบบ)
Project Mimir เป็นระบบ AI แพลตฟอร์มแบบ Multi-Tenant ที่ให้ผู้บริหารระบบ (SuperAdmin) สามารถแบ่งแยกพื้นที่ทำงาน (Workspace) ให้กับแต่ละ Tenant (หรือโปรเจกต์) ได้อย่างเป็นอิสระ โดยแต่ละ Tenant จะสามารถจัดการโมเดล AI (LLM), Vector Database (RAG) และทดสอบระบบผ่าน AI Playground ของตนเองได้โดยไม่ข้องแวะกับข้อมูลของ Tenant อื่น

ในคู่มือฉบับปรับปรุงนี้ จะนำเสนอรูปแบบการใช้งานตั้งแต่การล็อกอิน ตลอดจนถึงการประเมินคุณภาพของ AI ในขั้นตอนสุดท้าย เพื่อให้ผู้ใช้งานสามารถทำตามได้อย่างกะทัดรัดและเข้าใจง่าย

---

## 2. Getting Started (การเริ่มต้นใช้งาน)
### 2.1 การเข้าสู่ระบบ (Login)
หน้าจอล็อกอินถูกออกแบบมาให้เรียบง่ายและปลอดภัย รองรับการเข้าถึงสำหรับทั้ง SuperAdmin และ Tenant User
1. เปิดเบราว์เซอร์และเข้าไปที่ `http://localhost:3000` (หรือ URL ของเซิร์ฟเวอร์ที่ติดตั้ง)
2. กรอก **Email** และ **Password** ที่ได้รับมอบหมาย
   - *สิทธิ์ SuperAdmin สำหรับจัดการทุก Tenant (เช่น `admin@superadmin.com`)*
   - *สิทธิ์ Admin ของ Tenant เฉพาะเจาะจง (เช่น `admin@mimir.local`)*
3. กด **Sign In** เพื่อเข้าสู่ระบบ หากรหัสผ่านถูกต้อง ระบบจะพาคุณเข้าสู่หน้าจอ Dashboard อัตโนมัติ

## 3. Core Features & Usage (ฟีเจอร์หลักและการใช้งาน)

### 3.1 การจัดการพื้นที่ทำงาน (Tenant Management) - เฉพาะ SuperAdmin
ฟีเจอร์นี้สงวนไว้สำหรับผู้ดูแลระบบระดับสูงสุด ใช้ในการสร้างพื้นที่ทำงานใหม่ให้กับลูกค้าหรือโปรเจกต์ใหม่ เพื่อแยกข้อมูล (Data Isolation) ให้เป็นสัดส่วนชัดเจน
1. ล็อกอินด้วยบัญชีระดับ SuperAdmin
2. ที่แถบเมนูด้านซ้ายมือให้คลิก **Tenants**
3. ระบบจะแสดงตารางพื้นที่ทำงานทั้งหมดที่มีในหน้าต่างปัจจุบัน
4. **การสร้าง Tenant ใหม่:** กดปุ่ม **+ Add Tenant** มุมขวาบน จะมีหน้าต่างขึ้นมาให้ตั้งชื่อ, อีเมลแอดมินคนแรก, และทางเลือกว่าจะสร้าง Dedicated Vector DB หรือไม่
5. การกระทำนี้จะสร้าง Rule พื้นฐาน และแจกจ่าย Setting แบบอัตโนมัติตามโครงสร้าง

![หน้ารายการจัดการพื้นที่ทำงาน (Tenant Management)](./images/mockups/mock_tenant.png)

### 3.2 การนำเข้าและติดตามข้อมูล (Data Pipeline & Sources)
ระบบ Mimir สามารถประมวลผลข้อมูลเอกสารหรือข้อมูลเกมเพื่อสอน AI อัตโนมัติ ผ่าน 4 ขั้นตอน: (Sources -> Generating -> Pending QC -> Vectorized)
1. ไปที่เมนู **Sources** ระบบจะแสดงตัวบอกสถานะ (Pipeline Stepper) และรายการ Sources ที่เชื่อมต่ออยู่
2. กดปุ่ม **Run Pipeline** ระบบจะดึงข้อมูลจาก Source ที่ตั้งไว้ไปวิ่งบน Background Job
3. สังเกตป้ายกำกับ (Badge) ใต้ช่อง Status จะเปลี่ยนจาก `RUNNING` เป็น `COMPLETED` เมื่อเสร็จสิ้น

![หน้าจอควบคุมข้อมูลและดูสถิติ (Data Pipeline Dashboard)](./images/mockups/mock_dashboard.png)

***หมายเหตุ:** หากคุณต้องการจัดการคุณภาพข้อมูล (QC) อย่างละเอียด สามารถเข้าไปที่เมนู **Quality Control** ระบบจะให้คุณแยกแยะข้อมูลที่ AI วิเคราะห์แบบขัดแย้ง (Conflict) หรือซ้ำซ้อน (Duplicate) ได้ด้วยตัวเองผ่าน Kanban Board ก่อนส่งเข้า Vector DB ต่อไป*

---

### 3.3 การวัดผลและตรวจสอบตัวแทน (Agent Evaluations)
ฟีเจอร์สุดท้ายและสำคัญคือการทดสอบว่า AI ที่ผู้ใช้ปรับแต่ง/สอนเนื้อหาไป สามารถตอบคำถามได้ถูกต้องเที่ยงตรง (ไม่หลอน) มากแค่ไหน:
1. ไปที่เมนู **Evaluations** จากแถบเมนูด้านซ้าย
2. กดปุ่ม **✨ New Evaluation Wizard** เพื่อสั่งให้ระบบตั้งคำถาม (จากหน้า Golden QA) แล้วยิงเข้าหา AI Model โดยอัตโนมัติ
3. หน้าจอจะประมวลผลคะแนน 3 ด้าน: Accuracy, Completeness, และ Relevance แสดงผลเป็นเปอร์เซ็นต์
4. มีตารางเปรียบเทียบเชิงความร้อน (Heatmap Grid) เพื่อบอกว่าโมเดลไหนจับคู่กับ Agent ชุดไหนทำผลงานได้ดีที่สุด (เครื่องหมาย 🌟 หมายถึง Best overall score)
