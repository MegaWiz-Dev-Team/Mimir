# Asgard Internal Certificates

This directory contains the public certificates required to access the internal `.asgard.internal` network endpoints securely.

## ⚠️ SECURITY WARNING
**Never** store any private keys (`tls.key` or `.key` files) in this repository. This directory is strictly for PUBLIC certificates (`.crt` or `.pem`).

## 🔑 `asgard-root-ca.crt`

This is the Root Certificate Authority (CA) generated by K3s `cert-manager`. It is used to automatically sign all internal domains such as `mimir.asgard.internal`, `sso.asgard.internal`, and `api.asgard.internal`.

### How to Install (macOS)
If you want your Mac to fully trust all `.asgard.internal` domains, install this into your system Keychain:

```bash
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ./asgard-root-ca.crt
```
*(Alternatively, double-click the file, open Keychain Access -> System -> find "Asgard Private Enterprise Root CA" -> Get Info -> Trust -> Always Trust)*

### How to use with Node.js
If you are running scripts that need to communicate with Asgard endpoints, you can bypass Node's strict SSL verification by adding the following to your `.env`:
```
NODE_TLS_REJECT_UNAUTHORIZED=0
```
